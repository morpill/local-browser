# Security Policy

## Security Status

> **⚠️ This project is currently in an unstable development stage.**

LocalBrowser is under active development and has **not** yet reached a stable release. Security has not been fully reviewed, and vulnerabilities may still exist.

**Do not use LocalBrowser in production environments or for handling sensitive data.**

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

You can contact the maintainer by:

- GitHub Security reports

I aim to respond to all reports within **5 business days**.

## What to Include

Please include as much information as possible:

- A clear description of the vulnerability
- Steps to reproduce
- Expected behavior
- Actual behavior
- A proof of concept (if available)
- Affected version or commit
- Operating system and environment
- Any relevant logs or screenshots

## Report Policy

I follow a responsible disclosure process.

After receiving a report:

1. I will acknowledge receipt within **5 business days**.
2. I will investigate the issue.
3. If confirmed, I will work on a fix as soon as reasonably possible.
4. A security advisory may be published after a fix is available.

Please **do not publicly disclose** a vulnerability before it has been addressed or we have agreed on a disclosure timeline.

## Scope

This policy applies to:

- LocalBrowser
- Official releases
- The main development branch

Third-party libraries and dependencies should be reported to their respective maintainers unless the issue is caused by LocalBrowser's integration.

## Out of Scope

The following generally do not qualify as security vulnerabilities:

- Typos or documentation errors
- Feature requests
- Performance issues without a security impact
- Reports that require unrealistic attacker capabilities
- Issues caused solely by unsupported or modified environments

## Supported Versions

As the project is still under active development, **only the latest commit on the default branch is supported**.

Older commits and experimental branches may contain known vulnerabilities and are not maintained.

## Thank You

Thank you for helping make LocalBrowser more secure. Responsible vulnerability reports are greatly appreciated.
